# Plan 03-01: SignalR BoardHub — Real-Time Card Sync

## Goal

Complete the SignalR real-time pipeline. The `BoardHub` stub and `@microsoft/signalr` package are already present; what is missing is (1) JWT query-string auth for WebSocket connections, (2) tenant membership verification in `JoinBoard`, (3) broadcast calls from card mutation endpoints, and (4) a frontend hook that opens the connection, registers event handlers, and invalidates TanStack Query cache on incoming events. When all five tasks are done, any card mutation by one workspace member will appear on every other connected member's board within 2 seconds, without a page reload.

## Tasks

### Task 1: Fix JWT auth for SignalR WebSocket connections

**Type:** edit
**File:** `api/Nodefy.Api/Auth/JwtConfig.cs`

**What:** Add `JwtBearerEvents` with an `OnMessageReceived` handler that reads `access_token` from the query string and sets it as the bearer token when the request path starts with `/hubs/board`.

**Why:** Browsers cannot set the `Authorization` header for WebSocket or SSE connections. SignalR automatically appends `?access_token=<token>` to the upgrade URL. Without `OnMessageReceived`, the JWT middleware ignores this query-string token and all SignalR connections return 401 — the `[Authorize]` attribute on `BoardHub` blocks every connection before any hub method runs.

**How:**

Inside `AddJwtBearer(opts => { ... })`, after the existing `opts.TokenValidationParameters = ...` block (currently ends at line 27), add:

```csharp
opts.Events = new JwtBearerEvents
{
    OnMessageReceived = context =>
    {
        var accessToken = context.Request.Query["access_token"];
        var path = context.HttpContext.Request.Path;
        if (!string.IsNullOrEmpty(accessToken) &&
            path.StartsWithSegments("/hubs/board"))
        {
            context.Token = accessToken;
        }
        return Task.CompletedTask;
    }
};
```

The closing `});` that ends `AddJwtBearer` stays in place — this block is inserted before it. No other changes to this file.

**Verify:** `dotnet build api/Nodefy.Api` compiles without errors.

**Done:** `JwtConfig.cs` contains `OnMessageReceived` event handler; build passes.

### Task 2: Add tenant verification to BoardHub.JoinBoard

**Type:** edit
**File:** `api/Nodefy.Api/Hubs/BoardHub.cs`

**What:** Rewrite `BoardHub` to inject `AppDbContext` via constructor DI, validate the calling user's pipeline membership in `JoinBoard` using the `IgnoreQueryFilters` bootstrap pattern, and abort the connection if the user is not a member. Keep `LeaveBoard` unchanged.

**Why:** The current stub (`Groups.AddToGroupAsync` with zero checks) allows any authenticated user with a valid JWT to join any pipeline's group, including pipelines in other tenants. The tenant membership check is the only authorization gate before adding a connection to a broadcast group.

**How:**

Replace the entire contents of `api/Nodefy.Api/Hubs/BoardHub.cs` with:

```csharp
using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.SignalR;
using Microsoft.EntityFrameworkCore;
using Nodefy.Api.Data;
using System.Security.Claims;

namespace Nodefy.Api.Hubs;

[Authorize]
public class BoardHub : Hub
{
    private readonly AppDbContext _db;

    public BoardHub(AppDbContext db)
    {
        _db = db;
    }

    public async Task JoinBoard(string pipelineId)
    {
        // Validate pipelineId is a well-formed Guid
        if (!Guid.TryParse(pipelineId, out var pipelineGuid))
        {
            Context.Abort();
            return;
        }

        // Resolve userId from JWT "sub" claim (set as NameClaimType in JwtConfig)
        var userIdStr = Context.User?.FindFirst("sub")?.Value
                     ?? Context.User?.FindFirst(ClaimTypes.NameIdentifier)?.Value;
        if (!Guid.TryParse(userIdStr, out var userId))
        {
            Context.Abort();
            return;
        }

        // Bootstrap tenant via pipeline (IgnoreQueryFilters — TenantMiddleware does not
        // run for SignalR connections; do NOT call tenantService.SetTenant() here)
        var pipeline = await _db.Pipelines
            .IgnoreQueryFilters()
            .FirstOrDefaultAsync(p => p.Id == pipelineGuid);
        if (pipeline is null)
        {
            Context.Abort();
            return;
        }

        // Verify the user is a member of the workspace that owns this pipeline
        var isMember = await _db.WorkspaceMembers
            .IgnoreQueryFilters()
            .AnyAsync(m => m.TenantId == pipeline.TenantId && m.UserId == userId);
        if (!isMember)
        {
            Context.Abort();
            return;
        }

        await Groups.AddToGroupAsync(Context.ConnectionId, $"pipeline:{pipelineId}");
    }

    public Task LeaveBoard(string pipelineId)
        => Groups.RemoveFromGroupAsync(Context.ConnectionId, $"pipeline:{pipelineId}");
}
```

Key points:
- `AppDbContext` is injected — SignalR hubs support constructor DI exactly like HTTP endpoints.
- `IgnoreQueryFilters()` is used for both `Pipelines` and `WorkspaceMembers` queries. Do NOT call `ITenantService.SetTenant()` inside the hub: the hub runs outside the HTTP middleware pipeline and the `TenantMiddleware` does not execute for SignalR connections. Direct `TenantId` equality checks replace the global filter.
- `Context.Abort()` terminates the WebSocket connection. This is the correct hub equivalent of `Results.Forbid()`.
- The group key `$"pipeline:{pipelineId}"` (string, not Guid) matches the broadcast calls in Task 3.

**Verify:** `dotnet build api/Nodefy.Api` compiles without errors.

**Done:** `BoardHub.cs` injects `AppDbContext`, `JoinBoard` performs pipeline lookup + membership check + `Context.Abort()` on failure, and `LeaveBoard` remains a single-line remove.

### Task 4: Create the signalr-token Next.js route handler

**Type:** create
**File:** `frontend/src/app/api/signalr-token/route.ts`

**What:** A GET handler that reads the Auth.js session, mints a short-lived JWT using `mintApiToken`, and returns `{ token }`. This is the endpoint that `useBoardRealtime`'s `accessTokenFactory` fetches to obtain a bearer token for the SignalR WebSocket upgrade.

**Why:** Browsers cannot set the `Authorization` header for WebSocket connections, so SignalR's JS client uses `accessTokenFactory` — a function called before each connection attempt that must return a JWT string. The factory must call a server-side route (not read from `localStorage` or a cookie) because `AUTH_SECRET` must never be exposed to the browser. This route follows the same session-read + `mintApiToken` pattern used in other server-side API routes.

**How:**

Create `frontend/src/app/api/signalr-token/route.ts` with this exact content:

```typescript
import { auth } from "@/auth"
import { mintApiToken } from "@/lib/api-token"
import { NextResponse } from "next/server"

export async function GET() {
  const session = await auth()
  if (!session?.user?.id) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 })
  }

  const token = await mintApiToken({
    sub: session.user.id,
    email: session.user.email ?? "",
  })

  return NextResponse.json({ token })
}
```

Notes:
- `auth()` is the Auth.js v5 server-side session reader (same pattern as all other Next.js route handlers in this codebase).
- `mintApiToken` is in `frontend/src/lib/api-token.ts` — it signs a HS256 JWT with `AUTH_SECRET`, issuer `nodefy-frontend`, audience `nodefy-api`, expiry `1h`. These values match `JwtConfig.cs` `TokenValidationParameters`.
- No `tenantId` is passed to `mintApiToken` here — the hub's `JoinBoard` method resolves the tenant from the pipelineId, not from the token. This is intentional: a single token is valid for any pipeline the user is a member of.

**Verify:** `curl -s http://localhost:3000/api/signalr-token` (with a valid session cookie) returns `{ "token": "<jwt>" }`. Without a session, returns 401.

**Done:** File exists and returns a valid JWT for authenticated users, 401 for unauthenticated requests.

### Task 5: Create useBoardRealtime hook and wire it into BoardShell

**Type:** create + edit
**Files:**
- `frontend/src/hooks/use-board-realtime.ts` (create)
- `frontend/src/components/board/BoardShell.tsx` (edit)

**What:** Create the `useBoardRealtime(pipelineId)` hook that manages the full SignalR connection lifecycle, and call it inside `BoardShell` alongside the existing `useBoard` call.

**Why:** The hook must be co-located with `useBoard` inside `BoardShell` (not at app root) so the connection is created only when a board is mounted and torn down on unmount. The hook needs `useQueryClient()` for cache invalidation and `useUIStore` to read `draggingCardId` (to skip invalidation during active drag — prevents DnD flicker per Pitfall 5 in the research doc).

**How — `frontend/src/hooks/use-board-realtime.ts` (create):**

```typescript
"use client"
import { useEffect, useRef } from "react"
import * as signalR from "@microsoft/signalr"
import { useQueryClient } from "@tanstack/react-query"
import { useUIStore } from "@/store/ui-store"

const BACKEND_URL = process.env.NEXT_PUBLIC_BACKEND_URL ?? "http://localhost:5000"

export function useBoardRealtime(pipelineId: string) {
  const qc = useQueryClient()
  const connectionRef = useRef<signalR.HubConnection | null>(null)

  useEffect(() => {
    const connection = new signalR.HubConnectionBuilder()
      .withUrl(`${BACKEND_URL}/hubs/board`, {
        accessTokenFactory: async () => {
          const res = await fetch("/api/signalr-token")
          const { token } = await res.json()
          return token as string
        },
      })
      .withAutomaticReconnect([0, 2000, 10000, 30000])
      .configureLogging(signalR.LogLevel.Warning)
      .build()

    // Invalidate board cache unless a drag is in progress.
    // draggingCardId check prevents flicker: if member A is mid-drag and receives
    // their own broadcast, invalidateQueries would update board state under the
    // active drag and cause dnd-kit to lose track of the dragged card.
    const invalidate = () => {
      if (useUIStore.getState().draggingCardId) return
      qc.invalidateQueries({ queryKey: ["board", pipelineId] })
    }

    // Register ALL handlers BEFORE connection.start() — required by @microsoft/signalr docs.
    // Event names MUST match exactly what CardEndpoints.cs broadcasts (PascalCase).
    connection.on("CardMoved",    invalidate)
    connection.on("CardCreated",  invalidate)
    connection.on("CardUpdated",  invalidate)
    connection.on("CardArchived", invalidate)

    // After reconnect: re-invoke JoinBoard because SignalR group membership is
    // connection-scoped and is lost when the WebSocket drops.
    // Also invalidate to catch any mutations that occurred during the disconnect window.
    connection.onreconnected(async () => {
      await connection.invoke("JoinBoard", pipelineId).catch(() => {})
      qc.invalidateQueries({ queryKey: ["board", pipelineId] })
    })

    const start = async () => {
      try {
        await connection.start()
        await connection.invoke("JoinBoard", pipelineId)
      } catch {
        // withAutomaticReconnect handles retries; initial failure is non-fatal.
      }
    }

    connectionRef.current = connection
    start()

    return () => {
      // Gracefully leave the group before stopping (best-effort — do not await)
      connection.invoke("LeaveBoard", pipelineId).catch(() => {})
      connection.stop()
    }
  }, [pipelineId, qc])
}
```
