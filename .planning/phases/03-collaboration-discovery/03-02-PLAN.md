# Plan 03-02: Search & Filters — Card Discovery

## Goal

Add full card search and filter capability. The backend gains a `GET /pipelines/{id}/cards/search` endpoint with PostgreSQL `ILIKE` title search and optional AND-combined filters (assignee, close date range, monetary value range). The frontend gains a per-pipeline `BoardFilter` Zustand state, a `FilterBar` UI component inserted into `BoardShell`, and query-switching logic in `useBoard` that fetches from the search endpoint when any filter is active and from the normal board endpoint when all filters are clear. Column aggregates always reflect the filtered set because filtering is server-side.

### Task 1: Add GET /pipelines/{id}/cards/search endpoint to PipelineEndpoints

Add a new endpoint registration inside `MapPipelineEndpoints`, after the existing `GET /pipelines/{id}/board` handler. Follow the same board GET pattern (lines 62–94 of the current file):

```csharp
// GET /pipelines/{id}/cards/search
// Query params (all optional, combine with AND):
//   q             — case-insensitive title substring match (ILIKE)
//   assigneeId    — exact match on AssigneeId (Guid)
//   minValue      — MonetaryValue >= minValue
//   maxValue      — MonetaryValue <= maxValue
//   closeBefore   — CloseDate <= closeBefore (ISO8601)
//   closeAfter    — CloseDate >= closeAfter (ISO8601)
app.MapGet("/pipelines/{id:guid}/cards/search",
    async (Guid id,
           string? q,
           Guid? assigneeId,
           decimal? minValue,
           decimal? maxValue,
           DateTimeOffset? closeBefore,
           DateTimeOffset? closeAfter,
           AppDbContext db,
           ITenantService tenant) =>
{
    // Bootstrap tenant (IgnoreQueryFilters pattern — same as board GET)
    var pipeline = await db.Pipelines
        .IgnoreQueryFilters()
        .FirstOrDefaultAsync(p => p.Id == id);
    if (pipeline is null) return Results.NotFound();

    tenant.SetTenant(pipeline.TenantId);

    var stages = await db.Stages
        .Where(s => s.PipelineId == id)
        .OrderBy(s => s.Position)
        .ToListAsync();

    var stageDtos = new List<StageBoardDto>();
    foreach (var stage in stages)
    {
        // Start from non-archived cards in this stage
        IQueryable<Card> query = db.Cards
            .Where(c => c.StageId == stage.Id && c.ArchivedAt == null);

        // Apply optional AND-combined filters
        if (!string.IsNullOrWhiteSpace(q))
            query = query.Where(c => EF.Functions.ILike(c.Title, $"%{q}%"));
        if (assigneeId.HasValue)
            query = query.Where(c => c.AssigneeId == assigneeId);
        if (minValue.HasValue)
            query = query.Where(c => c.MonetaryValue >= minValue);
        if (maxValue.HasValue)
            query = query.Where(c => c.MonetaryValue <= maxValue);
        if (closeAfter.HasValue)
            query = query.Where(c => c.CloseDate >= closeAfter);
        if (closeBefore.HasValue)
            query = query.Where(c => c.CloseDate <= closeBefore);

        var cards = await query
            .OrderBy(c => c.Position)
            .Select(c => new CardSummaryDto(
                c.Id, c.Title, c.Description, c.MonetaryValue,
                c.StageId, c.PipelineId, c.AssigneeId, c.CloseDate,
                c.StageEnteredAt, c.Position, c.ArchivedAt, c.CreatedAt))
            .ToListAsync();

        // Recompute aggregates on the filtered set only
        stageDtos.Add(new StageBoardDto(
            stage.Id, stage.Name, stage.Position,
            cards.Count,
            cards.Sum(c => c.MonetaryValue ?? 0m),
            cards));
    }

    var pipelineDto = new PipelineDto(pipeline.Id, pipeline.Name, pipeline.Position);
    return Results.Ok(new BoardDto(pipelineDto, stageDtos));
}).RequireAuthorization();
```

Implementation notes:
- `EF.Functions.ILike` maps to PostgreSQL's native `ILIKE` operator via Npgsql. The `q` value is passed as a parameterized SQL parameter — there is no raw SQL interpolation, so this is safe from injection. The `%` wildcards are part of the pattern string, not SQL syntax.
- `CardSummaryDto`, `StageBoardDto`, `BoardDto`, and `PipelineDto` are the same DTOs used by the existing board GET handler — check their exact names and constructor signatures in the current `PipelineEndpoints.cs` and use them identically. Do not create new DTO types.
- If the existing board endpoint uses a different method to project cards (e.g., a `ToCardSummaryDto(Card c)` private method), use the same method here rather than inline `Select`.
- All params are bound from the query string by ASP.NET Core minimal API parameter binding automatically. `DateTimeOffset?` binds from ISO8601 strings. `decimal?` and `Guid?` bind from their string representations.

**Why:** (a) The same security pattern used by `GET /board` applies here — all backend requests must go through Next.js route handlers that use `apiFetch` (which adds `X-Tenant-Id` and mints the JWT server-side). Filter state is NOT persisted to localStorage — it is session-only (not added to `partialize`).

**How — `frontend/src/app/api/pipelines/[id]/cards/search/route.ts` (create):**

```typescript
import { NextResponse } from "next/server"
import { apiFetch } from "@/lib/api"

export async function GET(
  req: Request,
  { params }: { params: Promise<{ id: string }> }
) {
  const { id } = await params
  const url = new URL(req.url)
  const workspaceId = url.searchParams.get("workspaceId")
  if (!workspaceId) {
    return NextResponse.json({ error: "workspaceId is required" }, { status: 400 })
  }

  // Forward all search params to the .NET endpoint, excluding workspaceId
  // (workspaceId becomes tenantId for apiFetch — not a backend query param)
  const qs = new URLSearchParams()
  for (const [k, v] of url.searchParams.entries()) {
    if (k !== "workspaceId") qs.set(k, v)
  }

  try {
    const data = await apiFetch(`/pipelines/${id}/cards/search?${qs}`, {
      tenantId: workspaceId,
    })
    return NextResponse.json(data)
  } catch (e: any) {
    return NextResponse.json({ error: e.message }, { status: 500 })
  }
}
```

Forwarded params: `q`, `assigneeId`, `minValue`, `maxValue`, `closeBefore`, `closeAfter`. These are passed verbatim as the query string to the .NET endpoint. No validation here — the backend validates them.

Replace the existing `useQuery` call with one that switches based on active filters:
```typescript
  const filterActive = isFilterActive(filters)
  const searchQs = filterActive ? buildSearchQs(filters) : null

  const { data: board = initialData } = useQuery<BoardData>({
    queryKey: filterActive
      ? ["board", pipelineId, "search", searchQs]
      : ["board", pipelineId],
    queryFn: async () => {
      const url = filterActive
        ? `/api/pipelines/${pipelineId}/cards/search?${searchQs}`
        : `/api/pipelines/${pipelineId}/board?workspaceId=${workspaceId}`
      const res = await fetch(url)
      if (!res.ok) throw new Error(await res.text())
      return res.json()
    },
    initialData: filterActive ? undefined : initialData,
    staleTime: 30_000,
  })
```

Key points:
- When `filterActive` is false, the query key is `["board", pipelineId]` — identical to before, so the existing cache (populated by the server-side initial load and by `invalidateQueries` from `useBoardRealtime`) is reused.
- When `filterActive` is true, a different key `["board", pipelineId, "search", searchQs]` is used, preventing the filtered result from polluting the main board cache.
- `initialData` is only passed for the non-filtered case. The filtered query has no `initialData` (it starts with a loading state).
- `moveMutation` remains unchanged — card moves always go to `PATCH /cards/{id}/move` regardless of filter state.

**How — `frontend/src/components/board/FilterBar.tsx` (create):**

```typescript
"use client"
import { useState, useEffect, useRef } from "react"
import { X, SlidersHorizontal } from "lucide-react"
import { Input } from "@/components/ui/input"
import { Button } from "@/components/ui/button"
import type { BoardFilter } from "@/store/ui-store"

interface FilterBarProps {
  pipelineId: string
  filter: BoardFilter
  onChange: (patch: Partial<BoardFilter>) => void
  onClear: () => void
  hasActive: boolean
}

export function FilterBar({ pipelineId: _pipelineId, filter, onChange, onClear, hasActive }: FilterBarProps) {
  // Debounce title search: update Zustand only 300ms after the user stops typing.
  // This prevents a new HTTP request on every keystroke.
  const [localQ, setLocalQ] = useState(filter.q)
  const debounceRef = useRef<ReturnType<typeof setTimeout> | null>(null)

  // Sync localQ if filter is cleared externally (e.g., onClear button)
  useEffect(() => {
    setLocalQ(filter.q)
  }, [filter.q])

  function handleQChange(value: string) {
    setLocalQ(value)
    if (debounceRef.current) clearTimeout(debounceRef.current)
    debounceRef.current = setTimeout(() => {
      onChange({ q: value })
    }, 300)
  }

  return (
    <div className="flex flex-wrap items-center gap-2 px-4 py-2 border-b border-border bg-muted/30">
      <SlidersHorizontal className="h-4 w-4 text-muted-foreground flex-shrink-0" />

      {/* Title search */}
      <Input
        placeholder="Buscar por título..."
        value={localQ}
        onChange={(e) => handleQChange(e.target.value)}
        className="h-7 w-48 text-sm"
      />

      {/* Assignee UUID filter */}
      <Input
        placeholder="ID do responsável"
        value={filter.assigneeId ?? ""}
        onChange={(e) => onChange({ assigneeId: e.target.value || null })}
        className="h-7 w-48 text-sm font-mono"
      />

      {/* Close date range */}
      <div className="flex items-center gap-1">
        <Input
          type="date"
          title="Fechar após"
          value={filter.closeAfter ? filter.closeAfter.slice(0, 10) : ""}
          onChange={(e) =>
            onChange({ closeAfter: e.target.value ? e.target.value + "T00:00:00Z" : null })
          }
          className="h-7 w-36 text-sm"
        />
        <span className="text-xs text-muted-foreground">até</span>
        <Input
          type="date"
          title="Fechar antes de"
          value={filter.closeBefore ? filter.closeBefore.slice(0, 10) : ""}
          onChange={(e) =>
            onChange({ closeBefore: e.target.value ? e.target.value + "T23:59:59Z" : null })
          }
          className="h-7 w-36 text-sm"
        />
      </div>

      {/* Monetary value range */}
      <div className="flex items-center gap-1">
        <Input
          type="number"
          placeholder="Valor mín"
          value={filter.minValue ?? ""}
          onChange={(e) => onChange({ minValue: e.target.value ? Number(e.target.value) : null })}
          className="h-7 w-24 text-sm"
          min={0}
        />
        <span className="text-xs text-muted-foreground">–</span>
        <Input
          type="number"
          placeholder="Valor máx"
          value={filter.maxValue ?? ""}
          onChange={(e) => onChange({ maxValue: e.target.value ? Number(e.target.value) : null })}
          className="h-7 w-24 text-sm"
          min={0}
        />
      </div>

      {/* Clear button — only shown when at least one filter is active */}
      {hasActive && (
        <Button
          variant="ghost"
          size="sm"
          onClick={onClear}
          className="h-7 px-2 text-xs text-muted-foreground"
        >
          <X className="h-3 w-3 mr-1" />
          Limpar filtros
        </Button>
      )}
    </div>
  )
}
```

Notes:
- `localQ` is local React state for the input value; `filter.q` is the debounced Zustand state that drives the query. The 300ms debounce in `handleQChange` prevents a fetch per keystroke.
- The assignee field accepts a raw UUID string. In v1, users must paste a member UUID. A member picker dropdown is a v2 improvement (out of scope for DISC-02).
- Date inputs emit ISO8601 strings with `T00:00:00Z` / `T23:59:59Z` suffixes so they bind correctly to `DateTimeOffset?` on the backend.
- `"use client"` directive is required because this component uses `useState`, `useEffect`, and event handlers.
- All text is Portuguese per the project's UI convention (Buscar, Limpar filtros, etc.).
