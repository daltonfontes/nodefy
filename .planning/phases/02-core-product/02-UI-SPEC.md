## Interaction Contracts

### Sidebar Toggle (D-03)

1. Toggle button: `ChevronLeft` (16px) when expanded, `ChevronRight` (16px) when collapsed. `aria-label` switches between `"Recolher barra lateral"` and `"Expandir barra lateral"`. Positioned at top of sidebar, full-width clickable row (height 40px).
2. On collapse: sidebar width animates from 240px to 48px with `transition-all duration-200 ease-in-out`. Pipeline names and section label fade out (`opacity: 0`). Board area expands to fill vacated space.
3. On expand: reverse. Pipeline names fade in after sidebar reaches full width.
4. Collapsed state: Zustand `ui-store.ts` field `sidebarCollapsed: boolean`. Persisted to `localStorage` with key `nodefy_sidebar_collapsed`.

### Pipeline Selection

1. Click pipeline → router.push to `/workspace/[id]/pipeline/[pipelineId]`.
2. Active pipeline: visually highlighted (left border + accent tint).
3. Zustand `activePipelineId` updates on route change (not on click, to avoid flicker during navigation).

### Inline Pipeline Rename (D-02)

1. Click `…` → DropdownMenu → "Renomear": pipeline list item transforms to an `<input>` pre-filled with current name.
2. Input focused immediately on render.
3. Save on Enter or blur: PATCH request. Optimistic: name updates immediately in sidebar.
4. Cancel on Escape: input reverts to original name.
5. API failure: name reverts + toast "Não foi possível renomear. Tente novamente."

### Inline Stage Rename (D-05)

Same pattern as pipeline rename. Column header stage name becomes an `<input>` on "Renomear" click.

### Stage Reorder (D-05 — drag column header)

1. dnd-kit `useSortable` applied to each column.
2. Drag handle: entire column header (48px strip) is the drag area.
3. During drag: source column ghost (opacity-50, dashed border outline), `DragOverlay` renders a solid card at 100% opacity.
4. Drop: PATCH `/stages/{id}/position` with new fractional index position. Optimistic: columns reorder immediately.
5. Rollback on failure: columns snap back to pre-drag order + toast "Não foi possível reordenar. Tente novamente."

### Card Drag-and-Drop (CARD-04)

1. dnd-kit `DndContext` wraps the board. Each column is a `useDroppable`. Each card is `useDraggable`.
2. Drag activation: `PointerSensor` with `activationConstraint: { distance: 4 }` (4px tolerance to prevent accidental drags on click).
3. During drag:
   - Source card position: renders a ghost placeholder (dashed border, same dimensions, `bg-muted opacity-60`)
   - Ghost card (following cursor): `opacity-90`, `shadow-lg`, `rotate-1` tilt, `scale-105`
   - Valid drop column: lane background `bg-primary/5`, border `border-primary`
4. On drop:
   - Zustand `draggingCardId` clears immediately
   - TanStack Query `useMutation` fires optimistic update: card moves to target column at computed fractional position
   - `stage_entered_at` reset tracked via mutation payload (backend handles it)
5. Rollback: if mutation `onError` fires, `queryClient.setQueryData` restores previous board state + toast "Não foi possível mover o card. Tente novamente." (3 second duration, dismissible)
6. Keyboard accessibility: dnd-kit keyboard sensor enabled — Tab to focus card, Space to lift, Arrow keys to navigate columns, Space/Enter to drop, Escape to cancel.

### Card Detail Side Panel

1. Click card → URL gains `?card={cardId}` → Sheet opens with slide-in animation (300ms ease)
2. Closing: click overlay, press Escape, or click × button (`aria-label="Fechar painel"`) → URL loses `?card=` param → Sheet slides out
3. Inline save for title: Enter key or blur → PATCH `/cards/{id}` — optimistic: heading updates immediately
4. Field edit commit: blur → PATCH → optimistic. Rollback: field reverts to previous value + toast
5. Deep link: if page loads with `?card={cardId}`, Sheet opens immediately on mount (no extra click)

### Delete Pipeline / Stage Confirmation

1. Destructive actions (delete pipeline, delete stage) always require confirmation via shadcn Dialog (not browser confirm)
2. Dialog: heading + body explaining data loss + context-specific cancel ("Manter pipeline" or "Manter estágio", outline variant) + "Excluir" (destructive variant)
3. "Excluir" button shows spinner while DELETE request is in-flight; after success: dialog closes, item removed from list

### Fractional Index Rebalance Threshold (Claude's discretion)

- Rebalance triggered server-side when any `position` value in a stage is `< 1e-9` or `> 1e15`
- Backend rebalances the full list with evenly spaced values and returns updated positions in the same response
- Frontend applies returned positions to the local query cache — no additional fetch needed
